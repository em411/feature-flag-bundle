# Design: Backport FeatureFlagBundle to Symfony 4.4 / PHP 7.4

**Date:** 2026-05-22
**Status:** Approved

## Goal

Produce a fork of `ajgarlag/feature-flag-bundle` that runs on **Symfony 4.4** and
**PHP 7.4**. The upstream bundle targets PHP 8.1+ and Symfony 6.4/7.4/8.0 and uses
PHP 8 attributes plus several post-4.4 Symfony APIs. This backport preserves the
public behaviour (feature checker, providers, annotation/tag-based feature
declaration, Twig functions, web profiler panel) while removing every feature
that cannot run on the target platform.

## Scope

**In scope:** core feature checker, `ChainProvider` / `InMemoryProvider`, feature
declaration via `@AsFeature` Doctrine annotation **and** service tags, Twig
`feature_is_enabled()` / `feature_get_value()` functions, web profiler data
collector and panel, full test suite, CI on PHP 7.4/8.0 × Symfony 4.4.

**Out of scope (dropped):** routing integration. Symfony 4.4 has no condition
services (`routing.condition_service`, Symfony 6.4+) and route `condition`
expressions are compiled into the dumped URL matcher with no container access,
so feature checks inside route conditions cannot be ported cleanly. The routing
config file and its loader hook are removed; the README documents a
kernel/controller-level alternative instead.

## Target dependencies (`composer.json`)

- `require`: `php: ">=7.4"`, `psr/container: ^1.0`, `doctrine/annotations: ^1.13`.
- `require-dev`: `symfony/framework-bundle: ^4.4`, `symfony/http-kernel: ^4.4`,
  `symfony/twig-bundle: ^4.4`, `symfony/yaml: ^4.4`, `symfony/var-dumper: ^4.4`,
  `symfony/phpunit-bridge: ^4.4`, `phpunit/phpunit: ^9.5`,
  `friendsofphp/php-cs-fixer: ^3.80`, `twig/twig: ^2.12|^3.0`.
- Symfony components pinned to `^4.4`; `tests/bin/pin-symfony-version.php` kept
  for the CI matrix.

## PHP 7.4 language downgrades

| PHP 8 feature (upstream)              | PHP 7.4 replacement                                            |
|---------------------------------------|-----------------------------------------------------------------|
| `#[AsFeature]` attribute class        | `@AsFeature` Doctrine annotation (`@Annotation`, `@Target({"CLASS","METHOD"})`) |
| `readonly` promoted properties        | plain `private` typed properties (PHP 7.4 supports typed properties) |
| Constructor property promotion        | explicit property declarations + assignment in the constructor body |
| `mixed` parameter/return types        | type removed from the signature, documented with `@param`/`@return mixed` |
| `throw` used as an expression in `fn()`| rewritten as a full `function () { throw ...; }` closure        |
| `string|false` union return type      | type hint removed (affects only the rewritten test kernel)      |

Doctrine annotations are **not repeatable on a single element** (unlike the
upstream `IS_REPEATABLE` attribute). No existing fixture stacks two `@AsFeature`
on one class or method, so there is no behavioural loss. The service-tag path
(`ajgarlag.feature_flag.feature`) remains repeatable for that edge case and is
documented as the way to declare multiple features on one class.

## Symfony 4.4 framework downgrades

### Bundle & extension
- `FeatureFlagBundle` extends `Symfony\Component\HttpKernel\Bundle\Bundle`
  instead of `AbstractBundle`. The `loadExtension()` method is removed; the
  `FeatureFlagExtension` is auto-discovered by Symfony's naming convention.
  `build()` continues to register `FeatureFlagPass`.
- `FeatureFlagExtension` extends
  `Symfony\Component\DependencyInjection\Extension\Extension` (unchanged) and
  loads XML config.

### Feature autoconfiguration
- `registerAttributeForAutoconfiguration()` (Symfony 5.3+) does not exist in 4.4.
  Replaced by an annotation scan executed at the start of
  `FeatureFlagPass::process()`: iterate `$container->getDefinitions()`, resolve
  each definition's class via `$container->getReflectionClass()` (which also
  registers the class file for cache invalidation), read `@AsFeature` on the
  class and its public methods with a `Doctrine\Common\Annotations\AnnotationReader`,
  and add the `ajgarlag.feature_flag.feature` tag with `feature` / `method`
  attributes — mirroring the upstream attribute callback logic, including the
  "method value on a method-level annotation" `LogicException`.
- Manually declared `ajgarlag.feature_flag.feature` tags continue to work and
  are merged with annotation-sourced tags.
- `registerForAutoconfiguration(ProviderInterface::class)` (exists since 3.3) is
  kept unchanged.

### Configuration files
- `config/*.php` use the PHP DSL functions `service()`, `tagged_iterator()` and
  `abstract_arg()`, none available in Symfony 4.4. All config is converted to
  **XML** (`config/feature_flag.xml`, `config/feature_flag_twig.xml`,
  `config/feature_flag_debug.xml`) and loaded with `XmlFileLoader`. XML is the
  version-stable, conventional format for a 4.4-targeted bundle.
- `config/feature_flag_routing.php` is **deleted**.
- The in-memory provider's feature argument is declared as an empty array in XML
  and overwritten by `FeatureFlagPass` via `setArgument('$features', ...)`.

### Availability checks
- `ContainerBuilder::willBeAvailable()` (Symfony 5.4+) is replaced with plain
  `class_exists(\Twig\Environment::class)` for the Twig integration. The routing
  check is removed entirely.

### Web profiler
- `FeatureFlagDataCollector::collect()` signature is matched to Symfony 4.4
  (`collect(Request $request, Response $response, \Throwable $exception = null)`
  — verified against the 4.4 `DataCollectorInterface`).
- The profiler Twig template (`templates/Collector/feature_flag.html.twig`) is
  adapted to Symfony 4.4 `@WebProfiler` macro signatures.

## Tests

- PHPUnit pinned to `^9.5` (supports PHP 7.4). PHPUnit 10+ attributes
  (`#[DataProvider]`) are converted back to `@dataProvider` annotations.
- `phpunit.xml.dist` is rewritten to the PHPUnit 9.5 schema (the PHPUnit 10
  `<source>` element replaced by `<coverage>`/`<filter>`).
- The functional test harness currently depends on Symfony **internal** test
  classes (`Symfony\Bundle\FrameworkBundle\Tests\Functional\AbstractWebTestCase`,
  `TestBundle`, `TestDumpExtension`) and 6.x/7.x framework config keys
  (`handle_all_throwables`, `storage_factory_id`, `collect_serializer_data`,
  `http_method_override` as a bool). It is rewritten as a **self-contained**
  harness:
  - a minimal `AppKernel` extending `Symfony\Component\HttpKernel\Kernel`,
    with no union return types and no dependency on Symfony test internals;
  - `FeatureFlagTest` extends `Symfony\Bundle\FrameworkBundle\Test\KernelTestCase`
    and uses `static::$container` (Symfony 4.4 has no `static::getContainer()`);
  - YAML fixtures rewritten against the Symfony 4.4 `framework` config schema;
  - `bundles.php` reduced to `[FrameworkBundle, TwigBundle, FeatureFlagBundle]`.
  - The error-message assertions are updated to refer to `@AsFeature` /
    annotation wording instead of the `#[...]` attribute wording.
- Unit tests (`FeatureCheckerTest`, provider tests, `TraceableFeatureCheckerTest`,
  `FeatureFlagDataCollectorTest`) keep their assertions; only PHP 8 syntax in
  the test bodies/fixtures is downgraded.
- Test fixtures (`ClassFeature`, `MethodFeature`, …) switch from `#[AsFeature]`
  to `@AsFeature` annotations.

## CI

- `.github/workflows/ci.yml` matrix → `php: ['7.4', '8.0']` × `symfony: ['4.4.*']`.
- `shivammathur/setup-php@v2` and `actions/checkout@v4` retained (both support
  PHP 7.4). The `pin-symfony-version.php` + `composer update --prefer-stable`
  steps are kept; the `php-cs-fixer check` step is kept.
- `.php-cs-fixer.dist.php` ruleset adjusted so the fixer does not introduce PHP 8
  syntax (target the PHP 7.4 migration set, drop PHP 8 migration rules).

## Cleanups bundled with the backport

- `FeatureCheckerInterface` has a stray `use Symfony\Component\FeatureFlag\Provider\ProviderInterface;`
  (an unrelated Symfony component) referenced only in a docblock `{@see}`. It is
  corrected to the bundle's own `Ajgarlag\FeatureFlagBundle\Provider\ProviderInterface`.

## Documentation

- `README.md` updated: supported versions (Symfony 4.4 / PHP 7.4), `@AsFeature`
  annotation usage replacing the attribute examples, removal of the routing
  section, and a short "gating routes manually" recipe (a controller check or a
  `kernel.controller` listener calling `FeatureCheckerInterface`).
- A `UPGRADE` / fork note records that this branch is a 4.4/7.4 backport of
  upstream `0.3.x`, what was dropped (routing), and the annotation
  non-repeatability caveat.

## Verification

PHP/Composer are unavailable on the host, so a **PHP 7.4 Docker image** is built
for local verification: `composer update`, `phpunit`, and `php-cs-fixer` all run
inside the container, giving a fast local feedback loop. The GitHub Actions CI
matrix (PHP 7.4 & 8.0 × Symfony 4.4) is the final gate after the branch is
pushed to the fork. The work is not complete until both the local Docker run and
CI are green.

## Note on lazy feature services

The upstream bundle declares each DI-built feature closure as a `lazy` `\Closure`
service. Symfony 4.4 cannot proxy `\Closure` (it is `final`) and lazy services
require `symfony/proxy-manager-bridge`. Instead, the backport adds a
`Provider\ServiceProvider` that holds a PSR-11 `ServiceLocator` plus a
`feature => [serviceId, method]` map and returns a closure that resolves the
service from the locator on first call — preserving true laziness with no
proxies. `InMemoryProvider` is kept unchanged for userland use and unit tests.

## Risks / open points

- The Symfony 4.4 functional-test framework config schema must be matched
  exactly; CI is the source of truth and the YAML fixtures may need iteration.
- The 4.4 web profiler template macros differ from upstream; the template may
  need iteration against the actual 4.4 `WebProfilerBundle`.
